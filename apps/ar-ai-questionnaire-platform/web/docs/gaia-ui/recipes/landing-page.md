# Recipe: GAIA UI Landing Page

Use this when the user explicitly asks for a landing page or product page.

## Components

- `NavbarMenu`
- `RaisedButton`
- `PricingCard`
- `WorkflowCard`
- `ChatDemo`
- `iPhoneMockup`
- Chart components or `StatRow`

## Structure

- First viewport: product name, concise offer, primary action, and a real assistant/workflow visual.
- Product proof: show actual GAIA-style workflows or chat, not generic feature cards.
- Use cases: group components around assistant, workflow, and productivity patterns.
- Pricing: use `PricingCard` when the page has plans or packaging.

## Notes

- Keep the headline literal and specific.
- Do not make a decorative-only hero.
- Keep the primary visual inspectable and relevant to the product.
